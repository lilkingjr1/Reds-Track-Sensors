How to Use / Best Practices:

First off, every sensor has object properties (accessible by double clicking the sensor), which include simple things like colors, hiding the mounting bracket, rotating the bracket, or hiding the wire.
As for placing, there are three primary ways you can place a sensor: Snap to Mount, Snap to Track, or Manual Placement.

Snap to Mount:

  1. Choose a mount you would like to use, and place it next to your desired section of track.
  2. Either:
    • Manually position, angle, and scale it to your liking on the track.
      • OR
    • Enable "Snapping to Track CoR" via the object properties, and adjust the Translation settings below to position the mount in the correct position for your coaster type. This may require trial and error to get it aligned perfectly. However, once you get it right, you can copy the values to any other mount you put down, and they will all align perfectly with the track.
  3. If the mount will rest on a Special Track (ie. transfer track), specify "Special Track to Move With" from the object properties so it will move properly with the track.
  4. Copy the default name of the mount from the "Name" box at the top (it will be used later).
  5. Choose a sensor you would like to use, and place it roughly near where you would like it to snap to the rail (this can be done with multiple sensors in various positions).
  6. Open the object properties of the sensor, and paste the name of the mount into the "Name of Mount to Snap To" field.
    This is all you need to do to get a sensor to perfectly snap to a mount.
    The Mounting Configs (located in /mounting_configs/) are responsible for each sensor type's position, angle, and bracket properties on each mount type, and can be tweaked if you so desire. Any settings in the object properties (ie. flip mounting bracket, hide wire, etc.) will override mounting configs.
  7. Lastly, if the sensor will rest on a Special Track, set the special track in object properties as well.

Snap to Track:

  1. Choose a sensor you would like to use, and place it roughly near the track you would like it to snap to.
  2. Disable mount snapping by opening object properties and making "Name of Mount to Snap To" completely empty.
  3. If the sensor will rest on a Special Track (ie. transfer track), specify "Special Track to Move With" from the object properties so it will move properly with the track.
  4. Enable "Snapping to Track CoR", and adjust the Translation/Rotation settings below to position the sensor in the correct position for your coaster type. This may require trial and error to get it aligned perfectly. However, once you get it right, you can copy the values to any other sensor you put down, and they will all align perfectly with the track.

Manual Placement:

  1. Choose a sensor you would like to use, and manually position, angle, and scale it to you liking, anywhere you would like.
  2. Disable mount snapping by opening object properties and making "Name of Mount to Snap To" completely empty.
  3. Also disable "Enable Snapping to Track CoR" if it isn't already disabled.
  4. If the sensor will rest on a Special Track (ie. transfer track), specify "Special Track to Move With" from the object properties so it will move properly with the track.

"But what the heck does that mean Red? Legalese scares me..."

No worries! Here's an easy break down to help you meet the requirements: 😊

  • First off, a great summary of what you can and can't do with the pack can be found here.
  • You are totally free to use this pack in any of your parks that you plan to share with other people (ie. Steam Workshop, forums, etc.)
    as long as you...
    • Keep the LICENSE file in-tact and present within the pack's files.
    • This includes when you package the park into a .nl2pkg file (however, the SCOs should automatically package it with them, so no worries about having to do it manually).
    • Crediting the pack and the author in your project or project's description is not technically required by the license, but is greatly appreciated.
  • The license requires that all source code must be disclosed and available. That technically means locked/encrypted .nl2pkg's break the license. However, I am willing to ignore this requirement as long as the pack and author are properly credited, visible outside of the locked package.
  • You may modify and improve any part of the pack you would like, as long as the license stays in-tact and changes remain available to the public. This includes locked/encrypted .nl2pkg's; meaning, no, modified works of this package are not allowed to be encrypted! If you have an addition you would like made to the original pack to be used in a private park, please feel free to contact me and I will gladly look into adding it.
  • Lastly, you can technically redistribute modified versions of this pack if you really want to, but I would much rather prefer you provide improvements to me so I can incorporate them into the official/original pack. You will be be credited too! I promise! 😉
